# STM32MP157 A7 端对接说明（队员B/A7专用）

> **本文件为 A7 端开发与 M4 端核间通信对接说明。**
> 仅包含 A7 偏向的 RPMsg 交互协议与指令格式。

---

## 一、职责说明

A7 端负责在 Linux 上通过 RPMsg 向 M4 端发送控制指令，并接收 M4 的确认与事件上报。

- 运行环境：OpenSTLinux
- A7 端软件：Python 或其他应用层程序
- 交互节点：`/dev/ttyRPMSG0`
- 消息格式：ASCII 字符串，必须以 `\n` 结尾

---

## 二、核间通信协议

### 2.1 M4 指令接收格式

- 每条命令为 ASCII 文本
- 必须以 `\n` 结尾
- M4 端会去除尾部换行符后解析内容

### 2.2 可发送指令列表

| A7 发送指令字符串 | M4 执行动作 | M4 ACK 返回 |
| ----------------- | ----------- | ------------ |
| `LED:R\n` | WS2812B 全部亮红色（疲劳警告） | `ACK:LED:R\n` |
| `LED:G\n` | WS2812B 全部亮绿色（专注模式） | `ACK:LED:G\n` |
| `LED:B\n` | WS2812B 全部亮蓝色（休息模式） | `ACK:LED:B\n` |
| `LED:OFF\n` | WS2812B 全部熄灭 | `ACK:LED:OFF\n` |
| `BEEP:1\n` | 蜂鸣器响 1 秒（持续警报） | `ACK:BEEP:OK\n` |
| `BEEP:3\n` | 蜂鸣器响 3 次，每次 500ms | `ACK:BEEP:OK\n` |
| `STATUS:OK\n` | 状态 LED 闪烁一次 | `ACK:STATUS:OK\n` |

### 2.3 未知指令处理

- M4 端收到未知指令时，不执行正常动作
- 应返回错误上报：`ERR:UNKNOWN_CMD\n`

### 2.4 M4 主动事件上报

| 触发条件 | M4 发送内容 | 含义 |
| -------- | ------------ | ---- |
| 物理按键 (KEY0/PG3) 被按下 | `EVENT:BTN_PRESS\n` | 通知 A7 端手动打断或强制解除警报 |
| 硬件初始化失败 | `EVENT:ERR_HAL\n` | 通知 A7 端出现严重错误 |

---

## 三、开发注意事项

- 确保每次写入 `/dev/ttyRPMSG0` 的指令都以 `\n` 结尾。
- 不要在字符串中加入额外空格或非 ASCII 控制字符。
- 读取回复时，建议按行读取并判断 `ACK:`、`ERR:` 或 `EVENT:` 前缀。
- 若 M4 端无响应，应检查 remoteproc 是否已启动并且 `/dev/ttyRPMSG0` 已打开。

---

## 四、推荐 A7 端测试流程

1. 打开 `/dev/ttyRPMSG0` 为读写模式。
2. 发送测试指令，如 `LED:R\n`。
3. 读取并验证回复是否为 `ACK:LED:R\n`。
4. 逐条验证 `LED:G`、`LED:B`、`LED:OFF`、`BEEP:1`、`BEEP:3`、`STATUS:OK`。
5. 监听 A7 端是否可接收 `EVENT:` 事件上报。

---

## 五、与 M4 端交互关键点

- A7 端不需要了解 M4 端内部实现，只需遵循协议字符串。
- M4 端有 ACK 机制，收到正确命令后会立即回复。
- 对于未知命令，A7 端应处理 `ERR:UNKNOWN_CMD\n`。
- M4 端可能在硬件失败时主动上报 `EVENT:ERR_HAL\n`。
